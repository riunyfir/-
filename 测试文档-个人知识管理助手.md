# 个人知识管理智能助手项目说明（测试文档）

## 1. 项目背景

在学习和开发过程中，个人会积累大量笔记、课程资料、技术文章与会议纪要。  
传统文件夹检索效率较低，无法快速完成“总结、问答、知识关联”。  
因此需要一个本地可运行的个人知识管理智能助手（PKM Assistant）。

## 2. 核心目标

- 支持多格式文档上传（TXT、MD、PDF、DOCX）。
- 自动完成文档解析、切块、向量索引。
- 自动生成摘要与关键要点。
- 基于 RAG 实现智能问答，并返回引用来源。
- 根据摘要和全文生成标签分类。
- 自动生成可视化思维导图。

## 3. 典型用户流程

1. 用户上传文档。
2. 系统提取文本并保存元数据。
3. 系统进行切块并建立向量索引。
4. 用户点击“自动总结”，得到摘要、要点和大纲。
5. 用户点击“生成标签”，得到主题标签。
6. 用户在问答页面提问，系统基于检索上下文回答。
7. 用户在详情页查看思维导图，快速浏览知识结构。

## 4. 功能需求

### 4.1 文档读取 Agent（DocReaderAgent）

- 根据文件后缀选择解析器。
- TXT/MD 直接读取文本。
- PDF 使用文本抽取方式读取各页内容。
- DOCX 读取段落并拼接。
- 对无内容或异常文件给出失败状态。

### 4.2 总结 Agent（SummarizeAgent）

- 输出 1~3 句简短摘要。
- 输出 3~8 条关键要点。
- 输出用于思维导图的 Markdown 大纲。

### 4.3 问答 Agent（QnAAgent）

- 接收用户问题。
- 检索 topK 相关文本块。
- 结合上下文生成答案。
- 返回引用信息：document_id、chunk_id、chunk_index、quote。

### 4.4 标签分类 Agent（TaggerAgent）

- 基于全文和摘要生成 3~8 个标签。
- 每个标签包含名称和置信度分数。
- 支持后续按标签筛选知识库。

### 4.5 思维导图 Agent（MindmapAgent）

- 将摘要/全文转换为层级大纲。
- 大纲需包含一级主题、二级主题和要点。
- 前端以 markmap 方式渲染。

## 5. 技术方案（示例）

- 前端：React + Vite + TypeScript + TailwindCSS
- 后端：FastAPI
- 数据库：SQLite（元数据、摘要、标签、会话）
- 向量库：Chroma（本地持久化）
- 本地模型服务：Ollama
- 推荐模型：
  - 生成模型：qwen2.5:7b-instruct
  - 向量模型：nomic-embed-text

## 6. 数据表设计要点

- documents：文档基础信息和处理状态。
- document_texts：文档全文。
- chunks：切块内容和位置。
- summaries：摘要、要点、思维导图大纲。
- tags 与 document_tags：标签及文档映射关系。
- chat_sessions 与 chat_messages：问答会话与消息记录。

## 7. 接口设计要点

- POST /api/files：上传文件并返回 document_id。
- POST /api/documents/{id}/process：解析、切块、向量化。
- POST /api/documents/{id}/summarize：自动总结。
- POST /api/documents/{id}/tag：标签生成。
- POST /api/chat：智能问答。
- GET /api/documents/{id}/mindmap：思维导图数据。

## 8. 非功能需求

- 本地离线可运行，不依赖云端服务。
- 接口响应应具备错误提示与容错处理。
- 文档重复上传可通过 hash 去重。
- 问答需尽量返回可追溯引用，降低幻觉风险。

## 9. 风险与改进方向

- 风险1：本地模型推理速度慢。  
解决：减少上下文长度、采用流式输出、优化模型选择。
- 风险2：复杂 PDF 抽取质量不稳定。  
解决：引入 OCR 或更强解析器。
- 风险3：向量检索命中率波动。  
解决：优化切块策略、增加重排（rerank）。

## 10. 里程碑计划

- M1：上传、解析、切块、入库。
- M2：向量检索与问答闭环。
- M3：自动总结与标签分类。
- M4：思维导图展示与交互优化。
- M5：演示脚本与验收文档。

## 11. 结论

该项目通过“文档读取 + 总结 + 问答 + 标签 + 思维导图”形成闭环，  
能够显著提升个人知识管理效率，并具备良好的可扩展性。
